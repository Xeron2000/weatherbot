<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI argv → `weatherbot.cli.main` | 非法命令和参数会直接影响执行分支 |
| JSON 文件 → `weatherbot.persistence` | 历史 market/state 文件可能缺少新字段或结构不完整 |
| Candidate/quote dict → `weatherbot.paper_execution` | 缺字段输入可能让订单意图和状态转移漂移 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-quick-f5r-01 | T | `weatherbot/persistence.py` | mitigate | 用直接测试锁定旧 market/state JSON 的 backfill 与默认字段补齐行为，避免模块化后写出不兼容结构 |
| T-quick-f5r-02 | D | `weatherbot/cli.py` | mitigate | 用 CLI 单测验证未知命令/参数显式 `SystemExit`，防止错误输入落入错误执行分支 |
| T-quick-f5r-03 | T | `weatherbot/paper_execution.py` | mitigate | 用纯函数测试固定 `build_passive_order_intent()` reason code 与 `apply_order_transition()` append-only 语义 |
| T-quick-f5r-04 | R | 测试入口 / 回归流程 | mitigate | 用 pytest-cov fail-under gate 阻止关键模块失测后仍然“绿灯” |
</threat_model>
