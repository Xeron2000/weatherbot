<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| committed `config.json` → runtime `_cfg` | 错误 profile merge 会直接改变扫描、风控、下单与 paper execution 的运行参数 |
| selected profile → import-time constants | `weatherbot/__init__.py` 在 import 时缓存配置，若 merge 时机不对会导致入口仍吃旧值 |
| environment → final config | `VISUAL_CROSSING_KEY` 必须在 profile merge 后保持最终覆盖，避免 profile/JSON 把 secret 配置顶掉 |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-quick-n5y-02 | D | `weatherbot/__init__.py` import-time config consumption | mitigate | 在测试中 reload 入口模块，直接断言 `BALANCE`、`YES_STRATEGY`、`NO_STRATEGY`、`RISK_ROUTER` 来自选中 profile 的最终配置。 |
| T-quick-n5y-03 | I | `vc_key` 覆盖顺序 | mitigate | 保持 `VISUAL_CROSSING_KEY` 在 merge 后最后覆盖，并用回归测试锁定 env-first 语义。 |
| T-quick-n5y-04 | E | invalid `strategy_profile` selection | mitigate | 对不存在的 profile 名显式抛错，但缺少 profile 字段时继续按旧顶层配置运行，兼容 brownfield 旧配置。 |
