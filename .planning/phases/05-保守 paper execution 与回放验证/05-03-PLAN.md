## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted execution logs → replay view | 本地日志被解释为 operator-facing 证据 |
| CLI arguments → replay filter | 用户输入决定读取哪笔订单/市场的回放 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-08 | R | replay renderer | mitigate | replay 只能读取 persisted `execution_events` / `order_history` / `paper_execution_state`，禁止在展示层重跑仿真导致审计漂移。 |
| T-05-09 | I | CLI replay output | mitigate | 输出显式展示 queue/touch/partial/cancel/fill 证据字段，避免只给最终结果造成误导。 |
| T-05-10 | T | replay filters | mitigate | 对 market/order 过滤参数做严格匹配与缺失提示，避免错误过滤导致操作者查看到错误订单。 |
</threat_model>
