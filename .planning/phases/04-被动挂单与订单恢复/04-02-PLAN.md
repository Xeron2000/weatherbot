| Boundary | Description |
|----------|-------------|
| persisted candidate/routing facts → order lifecycle | scan 会把 Phase 2/3 真相升级为主动订单动作 |
| quote_snapshot → fill/refresh decisions | 报价变化会触发成交、刷新或撤单 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | T | `active_order` lifecycle | mitigate | 所有 refresh/cancel/fill 只消费持久化 `candidate_assessments`、`route_decisions`、`quote_snapshot`，并把终态写回 `order_history`，避免状态漂移。 |
| T-04-05 | D | fill trigger in `scan_and_update()` | mitigate | fill 仅在 `ask <= limit_price` 且 quote 可用时发生；quote 缺失时保持 working，不允许猜测成交。 |
| T-04-06 | R | cancel/refresh reasons | mitigate | `status_reason` 使用固定 reason code 并由 pytest 锁定，如 `quote_repriced`、`candidate_downgraded`、`market_no_longer_ready`、`expired`。 |
</threat_model>
