<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| market JSON → restored runtime state | 重启时所有未终结订单都从持久化事实恢复 |
| run loop monitor → active_order resume | 监控分支会继续推进 unfinished order |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | T | restore helpers | mitigate | `restore_order_state_from_markets()` 只接纳 `planned`/`working`/`partial` 为 active，并把 terminal 状态隔离到 summary/history。 |
| T-04-08 | D | restart resume timing | mitigate | `run_loop()` monitor 分支显式调用 order monitor/resume helper，避免重启后长时间不处理 working order。 |
| T-04-09 | R | duplicated order recreation | mitigate | resume 优先消费现有 `active_order`，pytest 断言同 market restart 后不会新增第二笔 active order。 |
</threat_model>
